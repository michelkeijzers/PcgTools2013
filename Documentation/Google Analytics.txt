Fixed Window
============
Instead of the current windowing system in PCG Tools, where each file has a separate window, I think there is a much nicer display method.
It would be simpler to use and more clear to users.

Instead of one window per file, there is a GUI which has a left and right pane. Each pane shows one or a part of a file.
To handle more than two files, each pane has a tab that can show a file or a part from it. Also an identical file can be shown in two or more different panes.

How does this look like:

PatchId
=======
Now: using inheritance, setting within constructor.

New: using design strategy pattern.

How:
- Factory depending on model creating PatchIdService.
- ProgramIdService, CombiIdService derive from PatchIdService implementing IPatchIdService.
- IPatchIdService defines GetId receiving a bank ID and index, returning the ID.
- IPatchIdService implementors use a Program/CombiBankService derived from BankService, implementing IBankService.
- IBankService defines GetType returning the type of a bank (int, user, extended, GM) and function to return index/ID convertor (GetId, GetIndex).
